# The Colors 규칙

### Process

 * 마일스톤별로 처리할 요구사항 분류

 * 요구사항을 구현하기 위한 각각의 스텝이 피쳐

 * 피쳐별로 완료조건을 설정

 * 구현해야 하는 피쳐별로 이슈를 만들기

 * 이슈별로 PR을 보낼 것

---

### Issue Rule

* `[이름][프로젝트 - 요구사항] Feature명`

---

### Commit Rule

* `[행동(Create, Modified, Delete)] #IssueNumber 커밋내용`

	> 필요시에 상세내용 기재
